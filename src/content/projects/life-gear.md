## 일반 도구에서 전문 장비로

이 시스템은 운영 중인 Minecraft 서버의 채광, 농사, 도축, 낚시 도구를 하나의 성장 체계로 연결하기 위해 개발한 장비 시스템 플러그인입니다. 단순히 아이템에 인챈트를 추가하는 데서 그치지 않고 **각성부터 전문력 생성, 등급 상승, 주문서 강화, 손상과 복구, 소유자 귀속까지 장비의 전체 생명주기**를 구현했습니다.

미각성 도구에 각성석을 사용해 성공하면 장비가 플레이어에게 귀속되고 용도에 맞는 전문 능력과 초기 등급을 얻습니다.

- 곡괭이: 채광력
- 도끼와 괭이: 수확력
- 검: 도축력
- 낚싯대: 낚시력

각성된 도구는 이름만 바뀌는 커스텀 아이템이 아니라, 서버의 각 생활 콘텐츠가 공통 규칙으로 읽고 사용할 수 있는 전문 장비가 됩니다.

## 표시 문자열과 분리한 PDC 데이터 설계

장비 상태를 이름이나 설명 문자열로 판단하면 아이템 이름 변경, 로어 수정, 설정 갱신 과정에서 데이터가 손상될 수 있습니다. 핵심 상태는 Paper의 Persistent Data Container에 저장하고, 플레이어에게 보이는 이름과 설명은 이 데이터를 바탕으로 다시 생성하도록 분리했습니다.

- 각성 상태와 장비 등급
- 직업별 전문 능력 수치
- 주문서별 강화 단계
- 등급 상승 누적 수치와 개량 횟수
- 손상 여부와 강화 제한 상태
- 소유자 UUID

서버 설정이나 표시 형식이 달라져도 장비의 실제 상태는 유지됩니다. 플레이어가 접속하거나 보관함을 열 때 기존 장비의 표시와 효과를 현재 설정에 맞게 갱신하는 처리도 포함했습니다.

## 하락 없는 개량과 천장형 등급 성장

각성 장비에는 레어, 에픽, 유니크, 레전더리 등급이 적용되며 등급별로 전문력 생성 범위가 달라집니다. 개량석은 현재 등급 범위 안에서 전문력을 다시 추첨하되, 새 값이 기존보다 낮으면 원래 수치를 유지합니다. 성장 시도 후 장비가 오히려 약해지는 불합리한 결과를 방지한 것입니다.

기본·고급·직업 전용·정밀·승급 개량석마다 사용할 수 있는 장비와 등급, 재추첨 범위, 승급 보너스를 별도로 설정할 수 있습니다. 정밀 개량석은 해당 등급의 상위 범위에서만 수치를 생성하고 직업 전용 개량석은 다른 장비에 사용할 수 없도록 제한했습니다.

등급 상승에는 확률과 누적 횟수를 함께 적용했습니다.

```text
확률에 의한 조기 등급 상승
또는
누적 횟수 도달에 의한 확정 등급 상승
```

개량을 시도할 때마다 승급 누적 수치가 쌓이고 설정된 천장에 도달하면 다음 등급으로 확정 상승합니다. 사용한 개량석에 따라 한 번에 쌓이는 수치와 추가 승급 확률이 달라지며, 최고 등급에서는 불필요한 누적값을 자동 정리합니다. 등급 순서, 능력치 범위, 승급 확률과 천장은 설정 파일에서 조정할 수 있습니다.

## 네 가지 결과를 가진 주문서 강화

주문서는 D, C, B, A 등급으로 구분되며 등급마다 성공률과 최대 단계가 다릅니다. 강화 결과를 성공과 실패의 이분법 대신 네 가지 상태로 설계했습니다.

- 성공: 강화 단계 상승
- 유지: 현재 단계 유지
- 하락: 강화 단계 1단계 감소
- 손상: 장비가 손상 상태로 전환

상위 주문서일수록 최대 강화 단계와 성공률이 높고 하락·손상 가능성은 낮아집니다. 기본 설정에서 A등급 주문서는 손상되지 않도록 구성했으며, 플레이어 계급 혜택도 외부 서비스 API로 조회해 성공률에 반영합니다.

강화 수치는 설명에만 표시되지 않고 실제 플레이에 적용됩니다. 곡괭이와 수확 도구의 작업 속도, 도축 피해, 낚시 희귀 보상 확률과 입질 대기 시간을 장비 강화 단계에 맞춰 조정합니다. 전문력 체계와 역할이 겹치는 행운·약탈·섬세한 손길, 별도 소모 요소로 사용하지 않는 내구성·수선 인챈트는 차단해 성장 시스템 간 중복을 막았습니다.

## 하나의 규칙으로 연결한 생활 콘텐츠

장비 플러그인은 채광, 수확, 도축, 낚시 시스템이 함께 사용하는 기반으로 동작합니다. 채광력은 파편과 보상 계산에, 수확력은 성숙 작물의 보상에, 도축력은 전용 콘텐츠의 피해와 보상에 반영됩니다. 낚시에는 행운과 입질 속도 강화가 적용됩니다.

수확 과정에서는 미성숙 작물, 도구와 맞지 않는 작물, 수박·호박 줄기 파괴를 차단합니다. 다른 플레이어에게 귀속된 장비로 콘텐츠를 이용하는 것도 막습니다. 각 플러그인이 장비 데이터를 제각각 해석하지 않도록 공통 서비스 API를 제공해 서버 전체가 동일한 성장 기준을 사용하도록 했습니다.

## 손상과 대장장이 복구

강화 중 손상된 장비는 삭제되거나 능력치를 잃는 대신 효과 사용과 추가 개량이 잠깁니다. 플레이어는 대장장이 GUI에 장비를 올리고 비용을 지불해 복구할 수 있습니다.

- Vault 경제 시스템을 통한 게임 머니 차감
- ItemsAdder 커스텀 수리 재료 소모

Citizens가 있는 환경에서는 대장장이 NPC Trait으로 복구 화면을 열 수 있고, 설치되어 있지 않아도 GUI 자체는 정상 동작하는 선택적 연동 구조로 만들었습니다. GUI를 닫거나 작업을 취소하면 장비와 재료를 반환하며, 인벤토리가 가득 찬 경우에는 플레이어 위치에 안전하게 드롭합니다.

## 거래와 유실을 막는 귀속 장비 통제

각성 장비는 소유자만 사용할 수 있습니다. 직접 전달이나 바닥 드롭뿐 아니라 상자, 배럴, 셜커 상자, 번들, 호퍼, 공용 창고와 다른 플레이어 소유 창고로 이동하는 경로까지 검증합니다. 본인의 엔더 상자와 본인 소유 개인 창고에는 보관할 수 있도록 예외를 두었습니다.

일반 클릭, Shift 클릭, 숫자 키 교환, 드래그, 자동 이동처럼 아이템이 옮겨질 수 있는 여러 이벤트를 함께 확인해 우회 가능성을 줄였습니다. 이동을 막는 데서 끝나지 않고, 예기치 않은 GUI 종료나 가득 찬 인벤토리에서도 아이템이 사라지지 않도록 반환 경로를 설계했습니다.

## 플레이어 중심의 강화 작업대

각성, 주문서 강화, 도구 개량은 하나의 강화 작업대 GUI에서 이용합니다. 메인 화면에서 사용할 수 있는 도구와 재료 수량을 확인하고, 세부 화면에서는 적용 전 상태와 차단 사유를 본 뒤 작업을 실행합니다.

- 도구와 주문서 종류 불일치
- 미각성 또는 손상 장비에 개량석 사용
- 다른 플레이어에게 귀속된 장비
- 이미 최대 단계인 강화
- 등급이나 직업 조건에 맞지 않는 개량석
- 강화가 잠긴 특수 장비

조건에 맞지 않는 조합은 실행 전에 차단해 재료를 돌려주고, 실제 강화 결과가 실패일 때만 재료를 소비합니다. 사용자의 조작 실수와 시스템상 실패를 구분하도록 처리했습니다.

## API·이벤트 연동과 자동 검증

각성과 개량 성공 시 전용 이벤트를 발생시켜 퀘스트와 안내 시스템이 진행 상황을 감지할 수 있게 했습니다. 장비 강화와 생활 활동은 계약·회사 시스템의 진행도에 반영되고, 채광·도축·보관함·계급 시스템과는 서비스 API와 공통 데이터 규칙으로 연동됩니다. 장비의 숫자 변화가 독립된 기능으로 끝나지 않고 서버의 계약, 조직, 생활 콘텐츠 전체에 영향을 주는 성장 활동으로 이어집니다.

JUnit 5로 다음 항목을 자동 검증합니다.

- 주문서와 도구 종류 매칭 및 단계별 실제 효과
- 강화 하락 시 최소 단계 보장
- 전문력 재추첨 범위와 프리셋 식별
- 수확 가능한 작물과 도구 판정
- 미성숙 작물·줄기 파괴 차단
- 귀속 장비의 보관 허용 범위
- 내구도 보호 대상 판정
- 계약·회사 활동 연동

현재 준비된 **자동 테스트 33개가 모두 통과**합니다. 이 프로젝트를 통해 확률과 천장을 포함한 성장 설계, 실제 콘텐츠에 적용되는 능력치, 안전한 아이템 데이터 관리, 귀속 장비 이동 통제, 플러그인 간 API 연동과 자동 테스트까지 장비 시스템 전체를 구현했습니다.
